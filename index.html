Gumi

Simple jQuery plugin for creating custom styled form select elements and dropdowns with CSS.

View project onGitHub

Installation

Installing Gumi is relatively straight forward. Simply include both the jQuery and Gumi library files (e.g. jquery.gumi.min.js) using script html element(s), like so:


<link rel="stylesheet" type="text/css" href="/css/gumi.min.css"></link>
<script type="text/javascript" src="/js/jquery.gumi.min.js"></script>

HTML Structure

Once created, all dropdowns have a common HTML structure, like so:


<div class="gumi">
  <button>Please select an option</button>
  <ul>
    <li data-value="1">Option 1</li>
    <li data-value="2">Option 2</li>
    <li data-value="3">Option 3</li>
  </ul>
</div>

Simple Javascript Initialize

After you have the HTML written, now you can initialize the plugin:


<script type="text/javascript">
  $(function() {
    $('.gumi').gumi();
  });
</script>

Advanced Options

Gumi ships with a bunch of customizable options:

Javascript Initization Options
buttonClass Optional String value. Default value is btn-default. This class will get applied to all of the button elements during initialization.
buttonSelectedClass Optional String value. Default value is btn-selected. Gumi applies this class to the option that is selected in the list of options dropdown.
optionDisabledClass Optional String value. Default value is option-disabled. All li options that have the data-disabled="true" will have this class. Sometimes you may want disabled options to look a little different.
dropdownClass Optional String value. Default value is dropdown-default. Gumi will apply a unique class to your dropdown of options. All ul elements that wrap around the li options will have this class.
onChange Optional function callback. A change event fires when the user selects an option from the dropdown. This provides a way to hook into that change event and write your own custom functionality.
onOpen Optional function callback. When the user clicks on the button to display the list of options, Gumi fires a custom onOpen event to hook into. This event fires just before the options are displayed.

Data Attributes for the Button button
data-name Optional String value. This will name the select that is associated with your dropdown, giving you the flexibility of customizing the name of the select name that get's submitted with your form.
data-class Optional String value. Gumi will apply this class to the hidden select that is associated with your dropdown, in case you ever want to write some custom code to hook into it.

Data Attributes for Options li
data-value The value to be used for the form submit. If ommitted, Gumi will use the html string between the li tags
data-label String value. Defaults to the data-value. Sometimes you want the display label of the option to be different then the option value that gets submitted with your form. You can achieve this by setting a data-label on the option.
data-selected Boolean value. Defaults to the first option value in the list. When used, Gumi will recognize the first occurence of the data-selected attribute as the default selected option. This gives you the flexibility to pre-set the default value.
data-disabled Boolean value. Default value is false. If the value is set to true, the option in the dropdown will be visible but won't be selectable.
data-selectable Boolean value. Default value is false. If the value is set to true, the option will not appear to the user as an option to select in the dropdown. This is generally used as a first item in a list of options to give the dropdown a unique call to action label (e.g. "Please Select An Option").
